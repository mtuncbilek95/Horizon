# Horizon Engine

It's mediator based, context and system separated rendering engine.

### Engine - Context
Data holder that may be needed as a global accessor. Engine holds them and every system or context can access required contexts
through engine.

### Engine - System
Main control subjects (a.k.a subsystem) of the engine. It means that they may need to do some main thread work at some point. 
E.g WindowSystem needs to call ProcessEvents() on each while loop.


## How Rendering Works
Data oriented ECS and FrameGraph works in sync to control graphics related actions. The current design is single-threaded. The reason
behind is to design the flow without being drowned in the complexity. But after it become more robust, multi-threaded job system
will control every part of the entire rendering.

### Entity Component System
It's entt based ECS system but with extra layers. entt will be handling the entity/component/data correlation, then prepare
RenderableObject for FrameGraph.

### Frame Graph Mechanism
Executor of the rendering pipeline. This will take RenderableObject information from ECS. This RenderableObject information is being
generated by ECS, then Frame Graph gets it and execute the draw calls or use object info to handle draw calls with a correct order.
